# Support

## Getting Help

Use the project's issue tracker for:

- Bug reports.
- Documentation problems.
- Questions about expected CLI behavior.
- Feature requests.
- Retrieval quality examples.

For security issues, follow `SECURITY.md` instead of opening a public issue with
details.

## Before Opening an Issue

Please check:

- `README.md` for setup and CLI usage.
- `CONTRIBUTING.md` for development commands.
- `docs/adrs/` for architecture decisions.
- `docs/ddd/` for domain model terminology.
- Existing issues for duplicate reports.

## Useful Details

Include these details when relevant:

- OS and shell.
- Rust and Cargo versions.
- Exact command you ran.
- Whether you used the default mock embedder or `--features onnx`.
- A small sample document or evaluation dataset when it is safe to share.
- The observed output and expected output.

Do not share private engineering documents, secrets, proprietary code, or API
keys in public support requests.
