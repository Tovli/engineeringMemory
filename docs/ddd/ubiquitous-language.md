# Ubiquitous Language — tovli Domain Glossary

This glossary defines every term used in the tovli domain model.
Terms appear in the context files and diagrams with these exact meanings.
When the PRD uses a term, this glossary aligns with that usage.

> Source: derived from `docs/prd.md` §§1–18.

---

## Core Concepts

### Document
A source file ingested into the system. A Document has a stable identity
derived from its `sourcePath`. Its `contentHash` (SHA-256 of file bytes) is
the idempotency key — re-ingesting an unchanged Document is a no-op.
A Document may be active, modified (triggers re-ingestion), or soft-deleted.

### Chunk
A contiguous, semantically coherent fragment of a Document produced by the
chunking pipeline. A Chunk is the atomic unit of retrieval. It stores its
own `contentHash` so it can be individually detected as changed. Each Chunk
carries its position in the Document (`chunkIndex`), a heading breadcrumb
(`headingPath`), a short `preview`, and a `tokenCount`.

### ChunkingConfig
A value object that parameterises the chunking strategy for a given ingestion
run: `targetChunkTokens`, `maxChunkTokens`, and `overlapTokens`. Changing
this config invalidates existing Chunks and triggers re-chunking.

### Embedding
A dense numeric vector (an array of floating-point numbers) that represents
the semantic meaning of a Chunk in a high-dimensional space. An Embedding is
inseparable from the `modelName` and `dimension` that produced it. Two
Embeddings are compatible only when their `(modelName, dimension)` pairs
match.

### EmbeddingModelVersion
A value object: `{ modelName: string; dimension: number }`. This pair is the
unit of compatibility. All Embeddings in a single vector index must share the
same EmbeddingModelVersion.

### VectorIndex
The physical index structure inside RuVector/Postgres (e.g. an HNSW index)
created over a set of Embeddings that all share the same EmbeddingModelVersion.
Querying a VectorIndex with an Embedding from a different EmbeddingModelVersion
is prohibited.

### Query
A user's information need expressed as natural language text. A Query captures
the original question text, the `SearchMode` chosen, any `MetadataFilter`
applied, and the `EmbeddingModelVersion` used to embed it. Queries are
immutable once created and are used as the anchor for RetrievalRuns, Answers,
and Feedback.

### SearchMode
An enumeration: `vector | keyword | hybrid`. Determines which retrieval
strategy the Retrieval context uses.
- `vector`: cosine/L2 similarity against the VectorIndex.
- `keyword`: BM25-style full-text matching.
- `hybrid`: fusion of vector and keyword scores.

### MetadataFilter
A set of predicates applied to Chunk metadata at query time (e.g. `project`,
`tag`, `sourcePath`). Filters narrow the candidate set before or during scoring.

### RetrievalRun
A single execution of a retrieval strategy against a Query. Captures the
ordered list of RetrievalResults, the search strategy, latency, and any explain
data. A RetrievalRun is the observable unit of retrieval.

### RetrievalResult
One entry in the ranked output of a RetrievalRun. Contains the Chunk identity,
its rank position, similarity/relevance score, the source path, and a preview.
A RetrievalResult is a value object owned by a RetrievalRun.

### TopK
The maximum number of RetrievalResults to return for a given Query. A
configuration parameter, not a domain entity.

### Answer
A natural-language response generated by the LLM layer from the context
supplied by a RetrievalRun. An Answer is always grounded in retrieved Chunks —
it must carry at least one Citation. An Answer with zero Citations is invalid.
An Answer may carry a `noAnswerReason` when retrieval scores are below
threshold.

### Citation
A value object that binds an Answer to a specific Chunk: `{ chunkId, rank,
sourcePath, preview }`. Citations are the traceability mechanism that fulfils
the "Sources Are Mandatory" principle.

### PromptTemplate
A versioned template used to construct the LLM prompt from retrieved Chunks.
The version string is stored with each Answer log to enable regression testing
against prompt changes.

### EvalQuestion
A test case in the evaluation dataset. Contains an id, the question text,
optionally expected Chunk IDs, and optionally expected source file paths.
EvalQuestions are static artefacts — they are not modified by system runs.

### EvalRun
An execution of the full evaluation dataset against one SearchMode and TopK
configuration. Produces Hit@K and MRR metrics, plus per-question detail.
An EvalRun is the aggregate root of the Evaluation context.

### HitAtK
A boolean metric for a single EvalQuestion: did any expected chunk appear in
the top K results? Hit@1, Hit@3, Hit@5 are computed per EvalRun.

### MRR
Mean Reciprocal Rank. The mean of (1 / rank_of_first_correct_result) across
all EvalQuestions in an EvalRun. Zero when no relevant result was found.

### Feedback
A user-provided rating (good / bad) attached to a specific Chunk in the
context of a specific Query. Feedback is the raw signal for retrieval
quality analysis. It is captured, stored, and reported — but does not
automatically affect ranking until explicitly enabled.

### FeedbackReport
A derived view aggregating Feedback across many Queries: shows queries with
predominantly bad Chunks, frequently downvoted Chunks, and queries that
returned no useful results. Used to guide re-chunking or re-indexing decisions.

### ContentHash
A SHA-256 digest of the raw byte content of a Document or Chunk. Used as the
idempotency key for ingestion and change detection.

### HeadingPath
An ordered list of Markdown heading strings tracing the path from the document
root to the section that contains a Chunk (e.g. `["Architecture", "Layering
Rules", "Aggregates"]`). Preserved to provide retrieval context.

### IngestionRun
A single execution of the ingestion pipeline over a folder or file set. Records
counts of files scanned, Chunks created, Embeddings generated, files skipped,
and files with errors.

### ReEmbedCommand
A user-initiated command that regenerates all Embeddings for active Chunks
under a new EmbeddingModelVersion. Must be followed by an EvalRun to confirm
quality is preserved.

### SimilarityThreshold
A configurable minimum score below which a RetrievalResult is considered
unreliable. When all results in a RetrievalRun fall below this threshold,
the Answer Generation context must produce a no-answer response.

### ExplainPayload
A debug artefact attached to a RetrievalRun when the user requests explain
mode. Contains the query embedding provider, filters applied, per-result
scores, and ranking method rationale.

### ACL (Anti-Corruption Layer)
An adapter interface that translates between the domain model's ubiquitous
language and the API of an external system (RuVector/Postgres, an LLM
provider, or an embedding provider). The domain never imports external library
types — it only speaks to ACLs.
